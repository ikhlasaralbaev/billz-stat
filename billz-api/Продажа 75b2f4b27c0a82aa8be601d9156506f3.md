# Продажа

## Создать новую

Метод позволяет создать новую продажу в статусе “*черновик*” в указанном магазине *shop_id* и в нужной кассе 

**обязательно**

Список касс доступен во [вспомогательных методах](%D0%9C%D0%B5%D1%82%D0%BE%D0%B4%D1%8B%20c242f4b27c0a820d856c81c9179e4051.md)

```bash
curl -X POST --location 'https://api-admin.billz.ai/v2/order?Billz-Response-Channel=HTTP' \
--header 'platform-id: 7d4a4c38-dd84-4902-b744-0488b80a4c01' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer ...' \
_
```

В ответ приходит ID новой продажи в формате UUID, а так же номер продажи, отображаемый в BILLZ UI

Пример ответа

```json
{
	"session_id": "fd8d352a-6299-4489-8a6e-a5c037f3f65b",
	"status_code": 200,
	"id": "38c3920a-ffdd-4233-a823-8af3266f5f4f",
	"error": {
		"code": "",
		"message": ""
	},
	"data": {
		**"id": "38c3920a-ffdd-4233-a823-8af3266f5f4f",**
		**"order_number": "5632631379",**
		"order_type": "SALE"
	},
	"correlation_id": "3674f28e-7b96-4683-87c3-80892698f421",
	"topic": "v2.order_service.order.created"
}
```

## Добавить продукт

После того как черновик создан, в него нужно добавить товары. Метод поддерживает добавление одного товарного наименования за раз

Передать можно ID продукта в *product_id* и кол-во единиц в *measurement_value*

Если передать *use_wholesale_price = true*, то подсчёт будет идти по оптовой цене

Товар может быть привязан к конкретному продавцу или нескольким продавцам.

Параметр "response_type": "HTTP" отдаёт ответ в этом же запросе вместо вебсокета

*пример добавления товара* 

```bash
curl 'https://api-admin.billz.ai/v2/order-product/:order_id?Billz-Response-Channel=HTTP' \
  -H 'Authorization: Bearer ...' \
  -H 'Content-Type: application/json' \
  --data-raw '
  {
    "sold_measurement_value": 1,
    "product_id": "2ce40c63-4527-4898-a975-7c30c255fd50",
    "used_wholesale_price": false,
    "is_manual": false,
    "seller_ids": [
        "90219ac7-88b7-43f6-bed9-5c6761aefa71"
    ],
    "response_type": "HTTP"
}
  '
```

### Продажа по свободной цене

В некоторых магазинах на товарах может быть включена опция продажи по свободной цене. Это означает, что товару может быть в корзине как с меньшей, так и с большей ценой продажи.

Для этого служит отдельный метод POST v2/order-product/:id в который передаётся комбинация полей free_price и is_manual

**пример запроса**

```json
curl 'https://[api-admin.billz.ai](https://api-admin.billz.ai/v1/orders)/v2/order-product/:orderId' \
  -H 'authorization: Bearer ..' \
  --data-raw '
  {
    "sold_measurement_value": 1,
    "product_id": "1b5e3f0d-ad00-44e5-981a-2c88af848ba3",
    "used_wholesale_price": false,
    "use_free_price": true,
    "free_price": 35000,
    "is_manual": true,
    "seller_ids": ["aade9196-0eb4-4511-80c1-085c746a30f3"],
    "response_type": "HTTP"
}
  
  '
```

## Cкидки

На все товары в корзине можно посадить ручную скидку в процентах или указать новую сумму.

Если требуется указать скидку на конкретный товар, то его следует указать в параметре **product_id**

параметры

| **поле** | **Описание** | **примечание** |
| --- | --- | --- |
| discount_unit | тип скидки | PERCENTAGE или CURRENCY |
| discount_value | значение скидки |  |
| product_id | товар, для которого применяется скидка | Не обязательное |

пример с ручной скидкой **в процентах**: применится скидка в 20%.

```json
curl -X POST --location '[https://api-admin.billz.ai](https://api-admin.billz.ai/v1/orders)/v2/order-manual-discount/:id?Billz-Response-Channel=HTTP'\
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer ….' \
--data '{
	"discount_unit":"PERCENTAGE",
	"discount_value":20,
	"product_id":"31a4c573-6a24-499e-9201-fab51d343d63"
	}'
```

Пример **с новой ценой** с учётом скидки: товар получит новую цену в 10.000 сум

```json
curl -X POST --location '[https://api-admin.billz.ai](https://api-admin.billz.ai/v1/orders)/v2/order-manual-discount/:id?Billz-Response-Channel=HTTP' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer ….' \
--data '{
	"discount_unit":"CURRENCY",
	"discount_value":10000,
	"product_id":"6403481a-b146-4a3f-a02c-b463fa3b2e88"
}'
```

## Завершить продажу

Готовый к продаже черновик можно завершить как продажу используя этот метод. 

Вам потребуется передать ID метода оплаты в *company_payment_type_id,* и указать сумму оплаты в параметре *paid_amount*

Список метода оплат можно увидеть в методе *GET [company_payment_methods](%D0%9C%D0%B5%D1%82%D0%BE%D0%B4%D1%8B%20c242f4b27c0a820d856c81c9179e4051.md)*

Завершение новой продажи выполняется вместе со списком оплат по этой продаже.

Если необходимо добавить комментарии при продаже, то текст можно передать в поле **comment**

**Входящие параметры**

**id** — id трназакции

пример запроса

```bash
curl --request POST \
  --url https://api-admin.billz.ai/v2/order-payment/:id?Billz-Response-Channel=HTTP \
  --header 'Authorization: Bearer ...' \
  --header 'Content-Type: application/json' \
  --data '{
	"payments": [
		{
			"company_payment_type_id": "c6e84d9b-0ef0-4d5a-886d-147c74a9b051",
			"paid_amount": 10000,
			"company_payment_type": {
				"name": "Наличные"
			},
			"returned_amount": 0
		},
		{
			"company_payment_type_id": "fe95ebe4-ae02-47d2-8d72-3c030a8a1035",
			"paid_amount": 100000,
			"company_payment_type": {
				"name": "UzCard"
			},
			"returned_amount": 0
		}
	],
	"comment": "test123",
	"with_cashback": 0,
	"without_cashback": false,
	"skip_ofd": false
}'
```

**исходящая модель**

В ответ приходит тип продажи “SALE”

```bash
{
"order_type":"SALE",
"should_print_cheque":true
}
```

## Отложить продажу

Готовый к продаже черновик можно сохранить как “отложку” используя этот метод. 

В этом случае, забронируется указанное кол-во товарных единиц и они не будут доступны для продажи, пока отложку не завершат или удалят (как вручную, так и автоматически)

Вам потребуется передать GUID продажи, указать дату истечения отложки, а так же указать комментарий (если есть)

Отложку можно завершить как продажу в UI BILLZ или с методом ‣ 

```bash
curl -X POST --location 'https://api-admin.billz.ai/v2/order/create_postpone?Billz-Response-Channel=HTTP' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer ….' \
--data '{
        "order_id": "e9a593b4-b67d-478e-b851-66e7f6d7e22c",
        "comment": "",
        "time": "2024-07-16 06:47:30"
    }'

```
