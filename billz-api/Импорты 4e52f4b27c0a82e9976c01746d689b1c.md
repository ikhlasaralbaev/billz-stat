### Создание с товарами

Метод позволяет создавать импорты с новыми или старыми товарами. Метод работает асинхронно, при вызове возвращает correlation_id, ответ можно получить через **вебхук** (*для подключения вебхука нужно обратиться к разработчикам*). 

```bash
curl --location 'https://api-admin.billz.ai/v2/product-import/create-with-products' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer ...' \
--data '{
  "shop_id": "32216bdd-28c6-4fd4-923a-4e9dd8a9263f",
	"products": [
		{
			"id": "",
			"name": "Nike Air Force 1",
			"barcode": "20000003143",
			"sku": "NK-AF1-D23",
      "description": "Nike Air Force 1. Classical Shoes",
			"measurement_value": 1,
			"retail_price": 20,
			"supply_price": 10,
	    "wholesale_price": 15,
	    "measurement_unit_id": "2e77e0b3-f173-4f77-b39a-5ec7f36c501a",
	    "category_ids": [
	        "f31da3e2-cded-478f-abb3-8b6d72c43653"
	    ],
	    "brand_id": "3a862e16-9e10-4850-9fd1-abe28c2a19b9",
	    "supplier_id": "cf2bf476-2215-4d19-be78-e36ba9610408",
	    "product_custom_fields": [
	        {
	            "product_characteristic_id": "4a8b56ca-4fea-4454-8171-72226873665b",
	            "value": "Black"
	        }
	    ]
		}
	]
}'
```

Тело запроса

- shop_id - айди магазина
    - ⚡️[Список магазинов](%D0%9C%D0%B5%D1%82%D0%BE%D0%B4%D1%8B%20c242f4b27c0a820d856c81c9179e4051.md)
- products - список с товарами
    - id - айди товара в системе (передается в случае если товар уже существует)
    - name - наименование товара
    - barcode - баркод
    - sku - артикул
    - description - описание товара
    - measurement_value - импортируемое кол-во
    - retail_price - цена продажи для импорта (если отличается с текущей ценой в магазине, будет заменять текущую цену)
    - supply_price - цена поставки
    - wholesale_price - оптовая цена
    - measurement_unit_id - единица измерения
        - ⚡️[Список единиц измерений](%D0%9C%D0%B5%D1%82%D0%BE%D0%B4%D1%8B%20c242f4b27c0a820d856c81c9179e4051.md)
    - category_ids - категории товара
        - ⚡️[Список категорий](%D0%9C%D0%B5%D1%82%D0%BE%D0%B4%D1%8B%20c242f4b27c0a820d856c81c9179e4051.md)
    - brand_id - айди бренда
        - ⚡️[Список брендов](%D0%9C%D0%B5%D1%82%D0%BE%D0%B4%D1%8B%20c242f4b27c0a820d856c81c9179e4051.md)
    - supplier_id - поставщики товара
        - ⚡️[Список поставщиков](%D0%9C%D0%B5%D1%82%D0%BE%D0%B4%D1%8B%20c242f4b27c0a820d856c81c9179e4051.md)
    - product_custom_fields - список харакетристик товаров
        - ⚡️[Список характеристик товаров](%D0%9C%D0%B5%D1%82%D0%BE%D0%B4%D1%8B%20c242f4b27c0a820d856c81c9179e4051.md)

### Ответ с вебхука

когда операция завершается, то в подключенный вебхук возвращается полный объект с начальным corellation_id

Для корректности ответа достаточно ориентироваться на поле  "**status_code**", которое при успешном ответе равно **200**

Другие поля так же возвращаются, и их можно использовать по необходимости

**структура ответа**

```go
type Response struct {
	NoResponse bool        `json:"no_response,omitempty"`
	SessionID  string      `json:"session_id,omitempty" swaggerignore:"true"`
	StatusCode int32       `json:"status_code,omitempty"`
	ID         string      `json:"id,omitempty"`
	Error      Error       `json:"error,omitempty"`
	Data      CreateWithProductImportModel `json:"data,omitempty"`
	Topic      string      `json:"topic,omitempty"`
	CompanyID  string      `json:"company_id"`
}

type CreateWithProductImportModel struct {
	Request
	CorrelationID string               `json:"correlation_id"`
	Products      []ImportProductModel `json:"products"`
	ShopID        string               `json:"shop_id"`
	Errors        []ValidateError      `json:"errors"`
}
type Request struct {
	CompanyID string `json:"company_id" swaggerignore:"true"`
	SessionID string `json:"session_id" swaggerignore:"true"`
	UserID    string `json:"user_id" swaggerignore:"true"`
}

type ImportProductModel struct {
	ImportItemID        string                           `json:"import_item_id"`
	ID                  string                           `json:"id" swaggerignore:"true"`
	Name                string                           `json:"name"`
	SKU                 string                           `json:"sku"`
	Barcode             string                           `json:"barcode"`
	Description         string                           `json:"description"`
	BrandID             string                           `json:"brand_id"`
	CategoryIDs         []string                         `json:"category_ids"`
	MeasurementUnitID   string                           `json:"measurement_unit_id"`
	SupplierID          string                           `json:"supplier_ids"`
	SupplyPrice         float64                          `json:"supply_price"`
	RetailPrice         float64                          `json:"retail_price"`
	WholesalePrice      float64                          `json:"wholesale_price"`
	ProductCustomFields []*CreateProductCustomFieldModel `json:"product_custom_fields"`
	MeasurementValue    float64                          `json:"measurement_value"`
}

type CreateProductCustomFieldModel struct {
	ID                      string `json:"id" swaggerignore:"true"`
	ProductDetailID         string `json:"product_detail_id"`
	ProductCharacteristicID string `json:"product_characteristic_id"`
	Value                   string `json:"value"`
	Name                    string `json:"name"`
	FromParent              bool   `json:"from_parent"`
}
type ValidateError struct {
	Line    int    `json:"line"`
	Message string `json:"message"`
}
```

### Список

Метод позволяет получать список импортов в магазине.

```bash
curl -X 'GET' \
  'https://api-admin.billz.ai/v2/import?limit=10&page=1' \
  -H 'accept: application/json' \
  -H 'Authorization: Bearer ...'
```

Входящие параметры

- limit - макс товаров в странице
- page - номер страницы
- shops - список айди магазинов через запятую
- start_date - с даты
- end_date - по дате

### Создание одного товара

Если есть необходимость создать товар на лету, без явного импорта, то можно вызвать метод POST /v2/product

**Входящая модель**

| **Поле** | **Описание** |
| --- | --- |
| **barcode** | Штрихкод товара |
| brand_id | Идентификатор бренда |
| brand_name | Название бренда |
| category_ids | Идентификаторы категорий |
| company_id | Идентификатор компании |
| description | Описание товара |

| images | Изображения товара |
| free_price | Индикатор свободной цены |
| is_auto_delivery | Индикатор автоматической доставки |
| is_auto_tax | Индикатор автоматического налогообложения |
| is_divisible | Индикатор делимости товара |
| is_variative | Индикатор вариативности товара |
| measurement_type | Тип измерения |
| **measurement_unit_id** | Идентификатор единицы измерения |
| **name** | Название товара |
| **product_type_id** | Идентификатор типа товара |
| retail_price | Розничная цена товара |
| shipments.has_trigger | Индикатор наличия триггера в поставке |
| **shipments.measurement_value** | Количество товара в поставке |

| **shop_prices.shop_id** | Идентификатор магазина для цен |
| **shop_prices.retail_price** | Розничная цена товара в магазине |
| **shop_prices.supply_price** | Закупочная цена товара в магазине |
| shop_prices.wholesale_price | Оптовая цена товара в магазине |

| scale_plu | Код PLU для весового товара |

**Обязательные поля выделены жирным**
Для минимального создания товара, необходимо заполнить

- **barcode**
- **sku**
- **name**
- **product_type_id**
- **measurement_unit_id**
- **company_id**
- массив **shipments** с кол-вом в нужном магазине
- массив **shop_prices** с ценами во всех магазинах
