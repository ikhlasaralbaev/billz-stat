# Сертификаты

## Поиск подарочных карт

Endpoint: `GET /v1/gift-card/search`

Параметры запроса

| Имя | Тип | Описание |
| --- | --- | --- |
| code | string | штрихкод подарочной карты |
| status_list | string | разделенные через запятую статусы подарочных карт (NEW, ACTIVE, USED, CLOSED, EXPIRED) |
| page | int | Номер страницы |
| limit | int | Количество строк на странице |
| types | string | Тип подарочной карты (CERTIFICATE, VOUCHER) |
| use_type | string | Тип использования карты SINGLE, MULTIPLE |
| from_created_at | string | Дата в формате dd.mm.yyyy - дата создания от |
| to_created_at | string | Дата в формате dd.mm.yyyy - дата создания до |
| from_expire_at | string | Дата в формате dd.mm.yyyy - дата истечения от |
| to_expire_at | string | Дата в формате dd.mm.yyyy - дата истечения до |
| from_sold_at | string | Дата в формате dd.mm.yyyy - дата продажи от |
| to_sold_at | string | Дата в формате dd.mm.yyyy - дата продажи до |
| from_used_at | string | Дата в формате dd.mm.yyyy - дата использования от |
| to_used_at | string | Дата в формате dd.mm.yyyy - дата использования до |

**Пример запроса**

```bash
curl --location 'https://api-admin.billz.ai/v1/gift-card/search?page=2&limit=10&status_list=&types=CERTIFICATE&show_stats=true&from=190&use_type=SINGLE&from_created_at=01.12.2024&to_created_at=31.12.2024&from_expire_at=01.12.2024&to_expire_at=31.12.2024&from_sold_at=01.12.2024&to_sold_at=31.12.2024&from_used_at=01.12.2024&to_used_at=31.12.2024' \
--header 'Authorization: Bearer ...' \
--header 'Accept: application/json, text/plain, */*' \
--header 'platform-id: 7d4a4c38-dd84-4902-b744-0488b80a4c01'
```

**Параметры ответа**

| Имя | Тип | Описание |
| --- | --- | --- |
| sold_at | string | 2023-12-13T09:42:39.552065Z - дата продаже в формате utc |

| id | int | внутренний id карты |

## Создание карты

Endpoint: `POST /v1/gift-card/activate-list`

Параметры запроса, в заголовок обязательно вставляем --header 'Billz-Response-Channel: HTTP’ чтобы получить ответ на запрос

| Имя | Тип | Описание |
| --- | --- | --- |
| is_activated | bool | создание предактивированной карты (возможно только для ваучера, для сертификата всегда должно быть false) |
| gift_cards | []gift_card | список карт для создания |
| gift_card.code | string | штрихкод карты |
| gift_card.amount | float | номинал карты |
| gift_card.expire_period | string | dd.mm.yyyy - дата истечения карты |
| card_type | string | тип карты CERTIFICATE, VOUCHER |
| expire_type | string | тип истечения карты - ABSOLUTE |
| use_type | string | тип использования - SINGLE, MULTIPLE |

Пример запроса

```bash
curl 'https://api-admin.billz.ai/v1/gift-card/activate-list' \
  -H 'Authorization: Bearer ...' \
  -H 'platform-id: 7d4a4c38-dd84-4902-b744-0488b80a4c01' \
  -H 'Content-Type: application/json' \
  -H 'Billz-Response-Channel: HTTP'
  --data-raw '{"id":0,"is_activated":false,"gift_cards":[{"code":"1000000001","amount":10000,"expire_period":"31.12.2024"}],"card_type":"CERTIFICATE","expire_type":"ABSOLUTE","use_type":"SINGLE"}'

```

Параметры ответа

| Имя | Тип | Описание |
| --- | --- | --- |
| session_id | uuid | внутренний id сессии подключения |
| status_code | int | в случае успеха 200 |
| id | uuid | id сообщения ответа |
| error.code | string | внутренний код ошибки |
| error.message | string | сообщение об ошибке |
| data | int | id импорта подарочных карт |
| correlation_id | string | id запроса |
| topic | string | тип ответа |

Пример ответа

```bash
{
    "session_id": "46d0e6e8-542c-4dde-9b11-e255fec487cb",
    "status_code": 200,
    "id": "13ccd580-ee15-41d5-9110-29b28215eed6",
    "error": {
        "code": "",
        "message": ""
    },
    "data": 4574,
    "correlation_id": "13ccd580-ee15-41d5-9110-29b28215eed6",
    "topic": "v1.marketing_service.gift_card.activate_list.success"
}
```

## Добавление сертификата в продажу

Endpoint: `PUT /v1/order-gift-card/:order_id`

Параметры запроса, в заголовок обязательно вставляем --header 'Billz-Response-Channel: HTTP’ чтобы получить ответ на запрос. В query params передается uuid продажи

| Имя | Тип | Описание |
| --- | --- | --- |
| gift_card_id | int | внутренний id карты, можно получить через метод search |

Пример запроса

```bash
curl --location --request PUT 'https://api-admin.billz.ai/v1/order-gift-card/c7d83308-e92d-46bb-a036-db46482b170f' \
--header 'Authorization: Bearer ...' \
--header 'Accept: application/json, text/plain, */*' \
--header 'platform-id: 7d4a4c38-dd84-4902-b744-0488b80a4c01' \
--header 'Content-Type: application/json' \
--header 'Billz-Response-Channel: HTTP' \
--data '{"gift_card_id":4405215}'
```

## Оплата сертификатом

Endpoint: `POST v2/order-payment/:order_id`

Используется стандартный метод завершения оплаты только в тело запроса устанавливается тип оплаты сертификат и добавляется параметр gift_card_id

Параметры запроса. В query params передается uuid продажи

| Имя | Тип | Описание |
| --- | --- | --- |
| payments | []payment | массив типов оплат |
| payment.company_payment_type_id | uuid | внутренний id типа оплаты |
| payment.paid_amount | float | сумма оплаты |

Пример запроса

```bash
curl --location 'https://api-admin.billz.ai/v2/order-payment/a0b53785-c840-4533-a22f-bfe162b8e302' \
--header 'Authorization: Bearer ...' \
--header 'Accept-Language: ru' \
--header 'platform-id: 7d4a4c38-dd84-4902-b744-0488b80a4c01' \
--header 'Content-Type: application/json' \
--data '{
    "payments": [
        {
            "company_payment_type_id": "429121de-a7ba-4a01-9e3d-d65999778816",
            "paid_amount": 125000,
        },
        {
            "company_payment_type_id": "a89c2130-6083-4902-9493-100c31c3b559",
            "paid_amount": 10000,
            "gift_card_id": 4405184,
        }
    ]
}'
```

Параметры ответа

| Имя | Тип | Описание |
| --- | --- | --- |
| session_id | uuid | внутренний id сессии подключения |
| status_code | int | в случае успеха 200 |
| error.code | string | внутренний код ошибки |
| error.message | string | сообщение об ошибке |
| correlation_id | string | id запроса |
| topic | string | тип ответа |

Пример ответа

```bash
{
```
