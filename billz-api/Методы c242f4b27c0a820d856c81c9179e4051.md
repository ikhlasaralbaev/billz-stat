# Списание

Как и с любым процессом, списания товаров состоят из нескольких шагов

1. создать новый процесс списания, в котором ещё нет товаров
2. добавить к этому процессу те товары, что необходимо списания
3. завершить процесс списания

после завершения, товары попадут в очередь на списание и выполнятся в отложенном режиме

## Создание списания

**Входящая модель**

- Название
- причина списания (справочник причин списаний)
- ID магазина
- комментарий

**тело запроса**

```bash
curl 'https://api-admin.billz.ai/v2/write-off' \
 
  -H 'Authorization: Bearer .....' \
  -H 'Content-Type: application/json' \
  --data-raw '
  {
    "comment": "",
    "name": "Списание 2024.09.16 12:19",
    "reason_id": "d6ccea76-7412-48b0-bad3-e0bc0f41673f",
    "shop_id": "70919136-9481-4c7e-8581-685d750d67a9"
}'
```

В ответ приходит ID списания, которое вы используете далее

## Добавить товар к списанию

**Входящая модель**

- Название
- writeoff_measurement_value — кол-во твоаров к списанию
- shop_id — ID магазина
- product_id — ID продукта

:id — это ID списания

обратите внимание на действие POST

**тело запроса**

```bash
curl --request POST \
  --url https://api-admin.billz.ai/v2/write-off/:id/add \
  --header 'Authorization: Bearer ..' \
  --header 'Content-Type: application/json' \
  --data '{
	"shop_id": "70919136-9481-4c7e-8581-685d750d67a9",
	"product_id": "aba9d9ad-5d37-4836-81a4-9a60648d087e",
	"writeoff_measurement_value": 1
}'
```

## Отклонить списание

Отклоняет незавершённое списание

:id — это ID списания

обратите внимание на действие PUT

**запрос**

```bash
curl 'https://api-admin.billz.ai/v2/write-off/cancel/:id' \
  -X 'PUT' \
  -H 'Authorization: Bearer .....' \
  -H 'Content-Type: application/json' 
    
```

# Методы для работы с инвентаризацией

## Список инвентаризаций

**Входящая модель**

- Название
- company_id — ID компании
- search — поисковая строка
- shop_ids — фльтра по списку компаний, через запятую
- start_created_date — фильтр по дате начала инвентаризации С
- start_finished_date —  фильтр по дате начала инвентаризации До
- end_created_date — фильтр по дате завершения инвентаризации С
- end_finished_date — фильтр по дате завершения инвентаризации До
- status_ids — фильтр по списку статусов ин-ций, через запятую
- type — фильтр по типу инвентаризации
- limit — кол-во строк на
- page —  номер страницы

**тело запроса**

```bash
curl -X 'GET' \
  'https://api-admin.billz.ai/v2/stocktaking?company_id=17a36439-d140-454d-aedb-ae1585681db1&limit=10&page=1&search=12121&shop_ids=6f244db4-75d6-4fac-9841-14867d45e36a&start_created_date=2024-11-12&start_finished_date=2024-11-14&end_created_date=2024-11-19&end_finished_date=2024-11-22&status_ids=7cbb2295-559e-4b72-a3c1-11ac24dffc6b&type=full' \
  -H 'accept: application/json' \
  -H 'Authorization: Bearer ...'
```
